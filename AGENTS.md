# AGENTS.md

## 适用范围

本文件适用于当前仓库根目录及其子目录。

## 工作约束

- 始终使用中文简体回复。
- 修改前先阅读现有实现，优先保持 `AstrBot Python 插件 + Node sidecar` 的分层边界。
- 不要将账号、token、二维码会话、群号联调记录等本地敏感信息提交到仓库。
- sidecar 的本地运行数据仅允许留在 `sidecar/data/`，并保持被 `.gitignore` 忽略。
- 涉及 git 提交、push、历史改写时，默认使用非交互命令。
- `_refs/` 使用 git submodule 管理；如需更新引用，先在子仓库完成提交，再回到父仓库更新 submodule 指针。

## 分支合并规范

### 1. 分支命名

- `main` 作为稳定集成分支，只接收已验证完成的变更。
- 新功能使用 `feature/<name>`。
- 缺陷修复使用 `fix/<name>`。
- 文档调整使用 `docs/<name>`。
- 维护/重构使用 `chore/<name>`。

### 2. 开发流程

1. 从 `main` 拉出独立分支，单个分支只解决一个明确主题。
2. 变更完成后先本地验证，再提交。
3. 提交信息使用简短的 conventional commit 风格。
4. 分支必须先 `push`，再进入合并流程。
5. 合并回 `main` 后立即 `push main`，保证远端与本地一致。

### 3. 合并顺序

- 多个分支并行时，优先合并底层依赖，再合并上层业务分支，再合并文档/说明类分支。
- 若某次迭代涉及 sidecar、插件主逻辑和文档，建议顺序为：
  1. 先合并 `feature/sidecar`
  2. 再合并业务功能分支，如 `feature/paid_car_task`、`feature/free_gacha`
  3. 最后更新文档、版本号和 `_refs` 指针
- 如果分支之间有依赖，必须先合并依赖分支，再处理依赖方分支。

### 4. 子模块规则

- `_refs/xyzw_web_helper` 等子模块只在子仓库内完成修改、提交、推送。
- 父仓库只负责更新 submodule 指针，不直接改写子模块历史。
- 当子模块和父仓库同时变更时，必须先完成子模块提交，再回到父仓库提交指针更新。

### 5. 合并前检查

- 合并前必须确认工作区干净或仅包含预期变更。
- 合并前至少执行一次语法/静态校验。
- 合并后确认 `git status` 仅保留预期提交结果。
- 未得到明确要求时，不做历史改写、强推或 `reset --hard`。

## 目录说明

- `main.py`：AstrBot 命令入口与调度逻辑
- `storage.py`：插件本地状态与账号存储
- `sidecar/`：Node sidecar
- `_refs/`：依赖仓库引用信息
